# Roadmap SDD e IA - Nível 1
## Meta: Dominar fundamentos de SDD e compreender integração de IA em requisitos

**Duração**: 8 semanas

---

## Estrutura Semanal

### Semana 1-2: Fundamentos de SDD
- [ ] revisar documento SDD na wiki (leitura com exemplos)
- [ ] escrever à mão os 7 tipos de requisitos (escrita para fixação)
- [ ] criar lista de conceitos de SDD (INVEST, MoSCoW, etc.) - forma de lista, não flashcards
- [ ] ler 1 artigo sobre boas práticas de requisitos
- [ ] escrever 3 histórias de usuário de exemplo (à mão no Samsung Notes)
- [ ] fazer quiz de autoavaliação sobre tipos de requisitos

### Semana 3-4: Escrita de Requisitos Profunda
- [ ] escrever template completo de caso de uso (1 exemplo)
- [ ] criar 5 requisitos funcionais para um projeto fictício
- [ ] criar 3 requisitos não-funcionais (performance, segurança)
- [ ] definir critérios de aceitação para cada requisito
- [ ] revisar e refinar requisitos escritos anteriormente
- [ ] criar checklist de qualidade de requisitos

### Semana 5-6: Introdução a IA em Requisitos
- [ ] pesquisar: o que é IA generativa e como se aplica a software
- [ ] escrever à mão 5 exemplos de features com IA
- [ ] criar requisitos para uma feature simples com IA (ex.: resumir texto)
- [ ] identificar riscos e considerações éticas de IA em software
- [ ] ler 1 artigo sobre ética em IA
- [ ] fazer mapa mental de aplicações de IA em produtos

### Semana 7-8: SPIKE e Prática
- [ ] definir 1 SPIKE para testar uma ferramenta de IA
- [ ] executar o SPIKE e documentar resultados (escrita à mão)
- [ ] criar lista de prós e contras da integração de IA
- [ ] escrever primeiro esboço de requisitos com IA
- [ ] revisar todo o conteúdo das semanas anteriores
- [ ] organizar anotações por tema no Samsung Notes

---

## Questionário de Avaliação - Nível 1

Responda por escrito:

1. Quais são os 7 tipos de requisitos? Dê um exemplo de cada.
2. Escreva uma história de usuário completa com critérios de aceitação.
3. Quais são os critérios INVEST para uma boa história de usuário?
4. Liste 3 considerações éticas ao incluir IA em um produto de software.
5. Descreva uma feature com IA que você gostaria de implementar.

**Meta mínima**: Acertar 70% para avançar ao Nível 2
