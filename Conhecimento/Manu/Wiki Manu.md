## Como Aprendo

### Padroes Observados

1. **Aprende por estrutura**
   - Cria pastas organizadas (JavaScript, .NET, JiuJitsu, Mercado Financeiro)
   - Faz referências entre notas (links wiki)

2. **Aprende fazendo anotações passo a passo**
   - Exemplo: "Passo a passo do .map()"
   - Gosta de decompor conceitos em etapas

3. **Usa formato visual**
   - Headers com indicadores (🔹, 📌, ⚠️)
   - Tabelas comparativas (IEnumerable vs List)
   - Blocos de código

4. **Começa estrutura primeiro, preenche depois**
   - Muitas notas têm apenas links (esqueleto)
   - Provavelmente vai desenvolvendo conforme estuda

5. **Foco em aplicação prática**
   - Anotações são funcionais (ex: comandos Git)
   - Menos teoria, mais "como usar"

6. **Gosta de Roadmaps**
   - Estruturas de aprendizado com etapas claras e progresso mensurável
   - Checkboxes para acompanhamento de atividades semanais
   - Marcos de progresso com questionários de avaliação

### Leitura em Ingles

**Abordagem progressiva recomendada:**

1. **Primeira leitura** → Leia inteira em inglês, sem paradas, para entender o fluxo geral
2. **Segunda leitura** → Grife palavras-chave que se repetem
3. **Terceira leitura** → Traduza só os termos técnicos essenciais

**Para vocabulario geral (não técnico):**
- Grife e continue → Muitas vezes o significado fica claro no contexto do parágrafo
- Só busque se a palavra aparecer muitas vezes ou bloquear compreensão
- Use ChatGPT/Copilot para explicar palavras no contexto específico (não só tradução)

**Sugestão prática:** Primeira leitura livre → segunda leitura com dicionário só para palavras que impedem de entender a ideia central → terceira leitura para fixação

**Resumo por parágrafo:** Excelente para fixar o aprendizado. Depois de ler cada seção, anote em 1-2 frases o núcleo da ideia.
