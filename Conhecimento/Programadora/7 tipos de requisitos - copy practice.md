## Tipos com Exemplos (copie estes 6)

1. **Requisitos de Negócio** - Por que o projeto existe, objetivos
   - Exemplo: "Reduzir em 50% o tempo para marcar aulas"

2. **Requisitos de Usuário** - O que usuários precisam fazer
   - Exemplo: "O aluno deve conseguir agendar aula pelo app"

3. **Requisitos Funcionais** - O que o sistema faz
   - Exemplo: "O sistema deve enviar email de confirmação"

4. **Requisitos Não Funcionais** - Qualidade, performance
   - Exemplo: "O sistema deve responder em menos de 2 segundos"

5. **Regras de Negócio** - Restrições do negócio
   - Exemplo: "Desconto só aplica para pedidos acima de R$100"

6. **Requisitos de Dados** - Estrutura de dados
   - Exemplo: "CPF deve ter 11 dígitos"
   
7. **Requisitos de Interface Externa** - Integrações com outros sistemas
   - Exemplo: "O sistema deve Integrar com Stripe para pagamentos"
